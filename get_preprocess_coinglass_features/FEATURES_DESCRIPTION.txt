================================================================================
                    ОПИСАНИЕ ФИЧЕЙ (features_endpoints.json)
================================================================================

Все эндпоинты относятся к API Coinglass и предоставляют исторические данные
по фьючерсным рынкам криптовалют.

--------------------------------------------------------------------------------
1. OPEN INTEREST (Открытый интерес)
--------------------------------------------------------------------------------

Открытый интерес — это общее количество открытых фьючерсных контрактов, 
которые ещё не были закрыты. Рост OI при росте цены = бычий сигнал.
Падение OI при падении цены = медвежий сигнал.

• open_interest_history
  - Данные по конкретной бирже и паре (например, Binance BTCUSDT)
  - Параметры: exchange, symbol, interval

• open_interest_aggregated
  - Агрегированные данные со всех бирж
  - Параметры: symbol (базовый актив, например BTC), interval

• open_interest_stablecoin
  - OI по стейблкоин-маржинальным контрактам (USDT/USDC margin)
  - Параметры: exchange_list, symbol, interval

• open_interest_coin_margin
  - OI по коин-маржинальным контрактам (BTC margin, ETH margin и т.д.)
  - Параметры: exchange_list, symbol, interval

--------------------------------------------------------------------------------
2. FUNDING RATE (Ставка финансирования)
--------------------------------------------------------------------------------

Funding Rate — периодические платежи между лонгами и шортами для поддержания
цены фьючерса близко к спотовой цене.
Положительный FR = лонги платят шортам (бычий рынок)
Отрицательный FR = шорты платят лонгам (медвежий рынок)

• funding_rate_history
  - FR по конкретной бирже и паре
  - Параметры: exchange, symbol, interval

• funding_rate_oi_weight
  - FR, взвешенный по Open Interest (более точный показатель)
  - Агрегировано по всем биржам
  - Параметры: symbol, interval

• funding_rate_vol_weight
  - FR, взвешенный по объёму торгов
  - Агрегировано по всем биржам
  - Параметры: symbol, interval

--------------------------------------------------------------------------------
3. LONG/SHORT RATIO (Соотношение лонгов и шортов)
--------------------------------------------------------------------------------

Показывает баланс между покупателями и продавцами на рынке.
Экстремальные значения могут указывать на перегретость рынка.

• global_long_short_account_ratio
  - Соотношение аккаунтов в лонгах vs шортах (все пользователи биржи)
  - Параметры: exchange, symbol, interval

• top_long_short_account_ratio
  - То же самое, но только для топ-трейдеров биржи
  - Параметры: exchange, symbol, interval

• top_long_short_position_ratio
  - Соотношение позиций (по объёму) топ-трейдеров
  - Параметры: exchange, symbol, interval

--------------------------------------------------------------------------------
4. NET POSITION (Чистая позиция)
--------------------------------------------------------------------------------

• net_position
  - Разница между лонгами и шортами в денежном выражении
  - Показывает, в какую сторону склоняется рынок
  - Параметры: exchange, symbol, interval

--------------------------------------------------------------------------------
5. LIQUIDATION (Ликвидации)
--------------------------------------------------------------------------------

Ликвидация — принудительное закрытие позиции при достижении цены ликвидации.
Массовые ликвидации часто приводят к каскадным движениям цены.

• liquidation_history
  - Данные по ликвидациям на конкретной бирже
  - Параметры: exchange, symbol, interval

• liquidation_aggregated
  - Агрегированные данные по ликвидациям со всех бирж
  - Параметры: exchange_list, symbol, interval

--------------------------------------------------------------------------------
6. ORDERBOOK (Книга ордеров)
--------------------------------------------------------------------------------

Показывает распределение лимитных ордеров на покупку (bids) и продажу (asks).
Дисбаланс может указывать на потенциальное направление движения.

• orderbook_ask_bids
  - Данные стакана по конкретной бирже
  - Параметры: exchange, symbol, interval

• orderbook_aggregated
  - Агрегированные данные стакана со всех бирж
  - Параметры: exchange_list, symbol, interval

--------------------------------------------------------------------------------
7. TAKER BUY/SELL VOLUME (Объём рыночных ордеров)
--------------------------------------------------------------------------------

Taker — это трейдер, который исполняет ордер по рыночной цене (market order).
Показывает агрессию покупателей vs продавцов.

• taker_buy_sell_volume
  - Объём рыночных покупок/продаж по конкретной бирже
  - Параметры: exchange, symbol, interval

• taker_buy_sell_volume_aggregated
  - Агрегированный объём со всех бирж
  - Параметры: exchange_list, symbol, interval

================================================================================
                           ПАРАМЕТРЫ
================================================================================

interval    — временной интервал: "1h", "4h", "1d" и т.д.
exchange    — конкретная биржа: "Binance", "OKX", "Bybit" и др.
exchange_list — список бирж (для агрегированных эндпоинтов)
symbol      — торговая пара (BTCUSDT) или базовый актив (BTC)

================================================================================
